# Music Room App 운영/유지보수 문서

작성일: 2026-05-26  
대상: 비개발자 운영자/관리자

## 1) 이 파일들의 역할

### `lib/screens/rooms/room_list_screen.dart`
- 앱 첫 화면(방 목록)입니다.
- 상단 사람 아이콘(계정 버튼)을 눌렀을 때 로그인/관리자/로그아웃 메뉴를 보여줍니다.
- 관리자면 관리자 대시보드(`/admin`)로 이동하고, 로그아웃도 여기서 처리합니다.

### `lib/core/router.dart`
- "누가 어느 화면에 갈 수 있는지"를 정하는 교통정리 파일입니다.
- 로그인 상태에서 `/login`으로 가면 다시 `/`로 보내는 규칙이 있습니다.
- 관리자 전용 화면(`/admin`) 접근 권한도 여기서 확인합니다.

### `lib/screens/auth/login_screen.dart`
- 로그인 입력 화면입니다.
- 뒤로가기 처리 방식이 잘못되면 크래시가 날 수 있는 민감한 화면입니다.

## 2) 꼭 알아야 할 개념 3개

### 1. 로그인 상태 (`user: true/false`)
- `user: true` = 이미 로그인된 상태
- `user: false` = 로그아웃 상태
- 이미 로그인된 상태에서는 로그인 화면으로 가려고 해도 라우터가 되돌릴 수 있습니다.

### 2. 권한(`role`)
- Firestore `users/{uid}` 문서의 `role`이 중요합니다.
- `role == 'admin'`이면 관리자 메뉴/화면 접근 가능
- 문자열 오타(`amin`, `Admin`)가 나면 관리자 기능이 안 보입니다.

### 3. 화면 이동 방식 (`go` vs `push`)
- `context.go('/...')`: 현재 화면을 교체 이동 (메인 전환에 적합)
- `context.push('/...')`: 화면을 위에 쌓아 이동 (로그인 같은 보조 화면)
- 이 둘을 잘못 섞으면 뒤로가기 에러가 발생할 수 있습니다.

## 3) 내가 수정하면 위험한 부분

- `role == 'admin'` 비교 문자열
- 로그아웃 후 이동 흐름(`signOut` -> `/login`)
- 로그인 화면 뒤로가기 방식(`Navigator.pop`로 바꾸면 재크래시 위험)
- 라우터의 `/login` 리다이렉트 규칙

## 4) 자주 나는 에러와 원인

### 에러 A
- 메시지: `You have popped the last page off of the stack`
- 원인: 더 이상 pop할 화면이 없는데 `Navigator.pop` 실행
- 해결: 로그인 화면 뒤로가기를 `context.go('/')`로 처리

### 에러 B
- 메시지: `!_debugLocked`
- 원인: 화면 전환(pop/go)이 같은 타이밍에 겹침
- 해결: 중복 전환 제거, pop 최소화

### 에러 C
- 증상: "로그인 버튼 눌러도 로그인 안 되는 것 같음"
- 실제 원인: 이미 로그인(`user:true`)이라 `/login` 접근 시 라우터가 `/`로 복귀시킴
- 해결: 계정 버튼을 로그인 버튼이 아닌 "계정 메뉴"로 동작

## 5) 다음 점검 체크리스트

- [ ] 앱 실행 로그에서 `user: true/false` 확인
- [ ] 사람 아이콘 클릭 시 상태별 메뉴가 맞게 뜨는지 확인
- [ ] 관리자 계정에서 "관리자 대시보드" 메뉴가 보이는지 확인
- [ ] 로그아웃 후 `/login`으로 이동되는지 확인
- [ ] 로그인 화면에서 뒤로가기 시 크래시가 없는지 확인

## 6) 구조 / 파일 역할 / 데이터 흐름

### 구조
- 홈(방 목록) -> 계정 버튼 -> 권한 조회 -> 메뉴 노출 -> 이동/로그아웃

### 데이터 흐름
1. 사용자가 사람 아이콘 클릭
2. `FirebaseAuth.currentUser`로 로그인 여부 확인
3. 로그인 상태면 `validateAndGetUser(uid)`로 사용자 문서 조회
4. `role == 'admin'`이면 관리자 메뉴 노출
5. 로그아웃 선택 시 `signOut()` 후 `/login` 이동

### 수정 주의사항
- 관리자 판별 문자열은 반드시 `'admin'`
- 로그인 상태에서 `/login` 강제 이동 로직 추가 금지
- 인증/메인 전환에 `Navigator.pop(context)` 남용 금지
- 라우터 리다이렉트 규칙을 임의로 삭제하지 말 것
